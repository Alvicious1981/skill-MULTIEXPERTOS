## 2. SCoT (Sequential Chain of Thought)

For complex reasoning tasks, the Manager and Experts must utilize Sequential Thinking:

1.  **Break Down**: Decompose the problem into atomic steps.
2.  **Order**: Arrange steps logically (dependencies first).
3.  **Execute**: Perform one step at a time.
4.  **Verify**: Check the result of each step before proceeding.
5.  **Synthesize**: Combine results into the final output.

*Trigger: When user request complexity > 5 or implies multi-stage reasoning.*

## 3. Clean Floor Protocol

Maintain a pristine workspace environment to prevent contamination and errors:

1.  **Status Check**: Always run `git status` before starting a task.
2.  **Reconnaissance**: Run `ls -R` or `tree` to understand the directory structure.
3.  **Isolation**: Work in feature branches or isolated directories when possible.
4.  **Cleanup**: Remove temporary files and artifacts after task completion.
5.  **No Ghost Files**: Ensure all created files are tracked or explicitly ignored.

*Mandate: Role 03 (DevOps) must enforce Clean Floor before any Deployment.*
